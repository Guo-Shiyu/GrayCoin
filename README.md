            为了避免数据冗余存储带来的不同步的问题, 让队伍与题均保存提交记录的引用计数指针, 函数查表.

            tuple, 与之配套的结构化绑定, 语法糖很方便    
            optional, 我仿佛在用 CPP 写 Rust  
            enum class + variant, 模仿 Rust 的带值枚举(tagged union) (附: 这让我想到了脚本语言对于动态类型的解决方案)  
            format / functor / lambda, 有点现代语言的样子了
